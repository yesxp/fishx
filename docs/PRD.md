# PRD — 鱼渔娱 MVP

> **鱼**（渔获）· **渔**（钓鱼）· **娱**（乐趣）
> 三个字，三种感受，一个 App。

---

## 产品定位

**一句话：** 鱼渔娱 — 让每次出钓都有鱼、有技、有趣。

| 维度 | 定义 |
|---|---|
| 目标用户 | 休闲钓鱼爱好者（每周钓 1-3 次） |
| 核心体验 | 拍一张渔获照，自动关联天气地点鱼种，出现在钓点地图上 |
| 首发平台 | H5（微信分享 / 浏览器访问） |
| 后续平台 | 微信小程序 → App（Uni-app 一套代码） |

---

## 底座：渔获照片

渔获照片是整个 App 的原子单位。不是"有个社区功能"，而是**每个功能都围绕渔获照片生长**。

一张渔获照片自动包含：

```
🐟 鱼种（AI 识别）+ 🌤️ 天气（自动抓取）+ 📍 地点（自动定位）+ ⏰ 时间
```

它同时是：
- 🐟 **鱼**的记录 — "我钓到了什么"
- 🎣 **渔**的数据 — 天气+地点+鱼种，喂给地图和天气页
- 🎉 **娱**的内容 — 出现在首页流、钓点气泡、附近推荐

---

## 导航结构

```
🏠 首页（渔获）  |  🌤️ 天时  |  🗺️ 钓点  |  👤 我的
```

**4 个 Tab，无独立社区入口。整个 App 就是社区。**

---

## 四个 Tab 详解

### 🏠 首页 — 渔获流

> 核心：看到"鱼" — 别人钓到了什么

| 区域 | 内容 | 互动方式 |
|---|---|---|
| 顶部 | 今日钓鱼指数一句话 + 定位城市 | 点击进天时详情 |
| 信息流 | 附近/最新的渔获照片瀑布流 | 点赞、评论、查看钓点 |
| 每张卡片 | 鱼种 + 钓点名 + 时间 + 天气小标 | 点击展开详情 |

**设计要点：**
- 打开 App → 第一眼看到的是**别人的渔获**，不是数据面板
- 瀑布流双列交错，小红书风格
- 每张卡片右下角：❤️ 点赞 + 💬 评论数
- 下拉刷新 + 无限滚动

---

### 🌤️ 天时 — 天气·潮汐·气压

> 核心：掌握"渔" — 今天能不能去、什么时候去最好

| 区域 | 内容 |
|---|---|
| 钓鱼指数 | 大数字 0-100 + 一句话建议 |
| 气压趋势 | 过去 12h + 未来 12h 折线图 |
| 核心数据 | 气温 / 气压 / 风力 / 天气 / 水温 |
| 最佳时段 | 今天最佳出钓时间窗口 |
| 逐小时鱼口 | 条形图展示每小时鱼活跃度 |
| 天时渔获 | **这个天气条件下，钓友们的渔获**（底部推荐） |

**设计要点：**
- 结论先行：先给"能/不能去"，再给数据
- 天时渔获让冷冰冰的天气数据有了人的温度

---

### 🗺️ 钓点 — 钓场·河流·湖泊

> 核心：发现"娱" — 去哪钓，看看那里怎么样

| 区域 | 内容 | 互动方式 |
|---|---|---|
| 地图 | 卫星/标准底图 + 钓点标记 | 滑动探索 |
| 钓点标记 | 气泡显示**最近一张渔获照片** | 点击查看详情 |
| 钓点卡片 | 钓点名 + 评分 + 最近渔获数 + 距离 | 导航 / 分享 / 标记打卡 |
| 钓点渔获流 | 这个钓点的所有渔获时间线 | 看到这里的钓鱼生态 |
| 搜索 | 按名称搜索钓点 | — |

**设计要点：**
- 标记不是空点，是**渔获照片气泡** — 地图本身就是内容
- 钓点详情页 = 这个地方的钓鱼故事（渔获时间线 + 评分 + 环境数据）
- 长按地图空白处 → 标记新钓点

---

### 👤 我的 — 个人中心

> 核心：我的"鱼渔娱"人生

| 区域 | 内容 |
|---|---|
| 头像 + 昵称 | 基本信息 |
| 统计卡片 | 总渔获 / 最常钓的鱼 / 去过几个钓点 / 最佳战绩 |
| 我的渔获 | 个人渔获时间线（全部记录） |
| 我的钓点 | 收藏/标记的钓点列表 |
| 设置 | 定位管理、通知、清除缓存、关于 |

## 核心用户流程

### 主流程：从看天气到晒鱼获

```
打开 App → 首页看到附近渔获（被激励）
  ↓
切到天时 → 看指数 72 "适合出钓" → 最佳时段 16:00-19:00
  ↓
切到钓点 → 地图看到附近标记 → 点开一个钓点 → 看到最近渔获
  ↓
决定去 → 导航
  ↓
到达 → 钓鱼
  ↓
钓到了 → 点 "+" → 拍照
  ↓
AI 识鱼 + 自动填入天气地点 → 确认保存
  ↓
渔获照片自动出现在：钓点地图气泡 / 首页流 / 天时页推荐
  ↓
其他钓友看到 → 点赞/评论 → 互动产生
```

### 辅助流程：回看渔获

```
我的 → 我的渔获 → 时间线浏览 → 点击查看详情
```

## 互动设计原则

**互动不是功能，是底座。** 贯穿每个模块的互动形式：

| 互动形式 | 在哪里出现 | 效果 |
|---|---|---|
| **点赞 ❤️** | 首页流、钓点详情、个人主页 | 低成本认可 |
| **评论 💬** | 首页流、钓点详情 | 一句话交流（限 100 字） |
| **渔获照片** | 首页流、钓点气泡、天时推荐 | 核心内容载体 |
| **钓点打卡** | 钓点详情页 | 标记"我来过" |
| **收藏 ⭐** | 钓点、渔获卡片 | 个人收藏夹 |

**不需要"发帖"、"关注"、"私信"。** 互动轻量化，降低用户负担。

## 竞品对比

| 维度 | 钓鱼佬 | 鱼渔娱 |
|---|---|---|
| 首页 | 功能超市（15+入口） | **渔获瀑布流**（内容即首页） |
| 社区 | 独立"钓友圈"Tab（论坛式） | **无独立社区，互动贯穿每个页面** |
| 地图 | 功能重（图层/合规/历史影像） | **渔获照片气泡**（地图即内容） |
| 天气 | 数据堆砌 | **结论先行 + 天气下的渔获推荐** |
| 记录 | 渔获相机（工具属性） | **拍照即发布**（记录=分享） |
| 设计 | 蓝白标准色，工具感 | **iOS 风格，浅色清爽，内容优先** |
| 体积 | 152MB | H5 秒开 |

## AI 钓鱼指数算法（V1 简化版）

| 因素 | 权重 | 理想范围 |
|---|---|---|
| 气压 | 30% | 1000-1025 hPa |
| 气温 | 20% | 15-30°C |
| 风力 | 15% | 1-3 级 |
| 温差 | 15% | 日温差 <8°C |
| 天气 | 10% | 多云/阴 |
| 降水 | 10% | 无雨/小雨 |

| 分数 | 等级 | 建议文案 |
|---|---|---|
| 80-100 | 极佳 | "今天爆护概率高，赶紧出门！" |
| 60-79 | 良好 | "适合出钓，选好钓点很重要" |
| 40-59 | 一般 | "可以试试，别期望太高" |
| 20-39 | 较差 | "今天鱼口可能不好，休息一天？" |
| 0-19 | 不宜 | "气压/温度不对，改天再去" |

## 技术选型

| 项 | 选型 | 理由 |
|---|---|---|
| 前端框架 | Uni-app (Vue 3) | 一套代码 → H5/小程序/App |
| UI | uni-ui + 自定义样式 | 基础组件走 uni-ui，视觉自定义 |
| CSS 参考 | uiverse.io | 按钮/卡片/加载动画灵感 |
| 地图 | 高德 JS API (H5) | 免费额度，卫星图+标记 |
| AI 识鱼 | 腾讯云图像识别 | ¥0.01/次，Top 20 淡水鱼 |
| 天气数据 | 和风天气 API | 免费版 1000 次/天 |
| 后端 | 云函数 / 轻量 Node | MVP 不需要复杂后端 |
| 数据库 | 云开发（uniCloud） | 零运维，免费额度 |
| 图片存储 | COS / 云存储 | 渔获照片 |

## 数据模型

```
用户表 (user)
├── id: string (openid)
├── nickname, avatar
├── created_at

渔获表 (catch) ⭐ 底座
├── id: string
├── user_id → user.id
├── fish_type: string (AI 识别)
├── confidence: float
├── photo_url: string
├── location: { lat, lng, spot_name }
├── weather: { temp, pressure, wind, condition }
├── caught_at: timestamp
├── like_count, comment_count
├── created_at

钓点表 (spot)
├── id: string
├── name: string
├── location: { lat, lng }
├── type: enum (野钓/黑坑/海钓/河流/湖泊)
├── rating: float
├── latest_catch_photo: string (缓存最近渔获图)
├── catch_count: int
├── created_by → user.id
├── created_at

点赞表 (like)
├── user_id, target_type (catch/spot), target_id
├── created_at

评论表 (comment)
├── id: string
├── catch_id → catch.id
├── user_id → user.id
├── content: string (≤100字)
├── created_at
```

## MVP 功能范围

| ID | 功能 | Tab | 优先级 |
|---|---|---|---|
| F1 | 首页渔获瀑布流 | 🏠首页 | P0 |
| F2 | 今日钓鱼指数 | 🏠首页顶部 | P0 |
| F3 | 天时详情（气压/时段/鱼口） | 🌤️天时 | P0 |
| F4 | 钓点地图 + 标记 | 🗺️钓点 | P0 |
| F5 | 渔获拍照 + AI 识鱼 | + 按钮 | P0 |
| F6 | 点赞/评论 | 贯穿 | P0 |

## 不做（V1 明确排除）

- 独立社区/论坛入口（互动贯穿不单独设模块）
- 关注/粉丝体系（V1 用钓点聚合代替）
- 私信功能
- 钓鱼百科/饵料大全（AI 问答替代）
- 会员/广告/商业化
- 潮汐/台风/月相（V2）
- 禁渔区合规提醒
- 多钓法切换（V1 综合指数）
